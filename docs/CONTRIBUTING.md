# Contributing to Pixi Environment Manager

We are open to Pull Requests! We love your input! We want to make contributing to this project as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Guidelines

### Pull Requests
1.  **Dogfooding**: Please test your changes by using them in your day-to-day workflow. This helps verify stability and catches edge cases.
2.  **Testing**: **All unit tests must pass** before submitting a Pull Request. Run `npm run test` locally to verify.
3.  **Code Style**: Run `npm run lint` to check for styling issues.

### Development Setup
For instructions on how to build, run, and test the extension locally, please see [DEVELOPMENT.md](DEVELOPMENT.md).
